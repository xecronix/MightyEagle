add_action_cb

include mightyeagle.e
namespace mighty_eagle
public function add_action_cb(mighty_eagle_t self, sequence action_tag, atom function_id)

Adds a callback added to the mighty_eagle_t to be fired when the parser finds action tags.

Parameters:
  1. action_tag The action tag associated with the a function
  2. function_id A routine_id to run when the specifice action_tag is found during parsing
Returns:

0

Comments:

The function identified by function_id needs to implement the following interface/function signature like the example below. 

function action_cb(mighty_eagle_t me, sequence tag, sequence template, map:map context)

The function is expected to return a string or an atom if there was an error. The context map is the current tag name/value pairs in the current context during parsing.

parse

include mightyeagle.e
namespace mighty_eagle
public function parse(mighty_eagle_t self, sequence template_str, map :map data)

Function looks for tags and actions in a template then calls the appropreite callback to handle replacing tags with actual data.

Parameters:
  1. template_str : This is the template or sub template to parse
  2. data : A map of data to use to replace tags with values.
Returns:

retval : string representaion of the template or sub template with all tags replace by values

Comments:

If a tag can not be found in either the callbacks or data, the function will insert the tag back into the return value.

Parsing rules:


{ opens a tag
= identifies that a tag is for substitution
@ identifies that a tag is for action callbacks
Example: {=
:} closes a tag
Spaces are allowed before and after tag name.
Action tags can contain tags or other action tags.
